Game definition

Battles of Antargis is a project to develop a real-time-strategy (RTS) game, which is a mixture of 3 three games:

  1. Powermonger
  2. Settlers
  3. Warcraft

The main purpose of this game is conquering a medieval-like world. While doing so with your hero(es) you meet friends and enemies. Enemies can be fought with your troopers that you can arm with weapons and armour. In some cases you'll have to build these weapons or new buildings. Another thing is that resources are quiet rare.

The deciding differences to normal RTS-games like Warcraft are:
  • Story-bound - the game is strongly story-bound. So you will be told some story while playing the game and not only in between the different levels
  • Only 2 unit-types - This game only provides 2 kinds of units: heroes and troopers. The latter are the normal people, which do their work and can be recruited by heroes. Each player has an avatar within the game. This one is the player himself. So if this avatar dies, the game (or the level) is lost. The player can however gain control over other heroes.
  • Restricted control - You can’t control your troopers directly. You can assign jobs to your heroes. They will then automatically control their troopers.

Architectural overview

As mentioned above Battles of Antargis is splitted into a c++ and a ruby part. The advantage of this approach are the shorter development-cycles. It's easier to restart a program after modifying some lines of ruby code instead of having to compile the whole xxxx-load before doing so.

So much for the reasons. Here's the logical structure of the game from bottom to the top:

  • C++ and libraries (OpenGL, libSDL, ...)
  • GUI-library named AntargisGUI
  • 3d-Engine
  • Game-Engine
  • Scripting-Interface to Ruby done with swig.
  • plain game-logic done in Ruby
  • scripting interfaces for AI and level-scripting
  • AI and levels' scripts
